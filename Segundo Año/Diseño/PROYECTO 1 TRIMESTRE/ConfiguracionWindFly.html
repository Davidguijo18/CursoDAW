Configuración de WildFly

1. Configurar el Archivo Principal (standalone.xml)

Edita el archivo principal de configuración para personalizar el servidor.

sudo nano /opt/wildfly/standalone/configuration/standalone.xml

En este archivo, puedes modificar:

  • El puerto HTTP (por defecto es 8080):
    • <socket-binding name="http" port="8080"/>
  • Configuraciones JNDI para bases de datos o servicios externos.

2. Configuración de Variables de Entorno

Configura las variables de entorno necesarias:

export JAVA_HOME=/ruta/a/jdk
export WILDFLY_HOME=/ruta/a/wildfly

Para que estas configuraciones sean permanentes:

sudo nano ~/.bashrc

Y añade las líneas anteriores al archivo.

3. Configurar HTTPS

Habilitar HTTPS es importante para conexiones seguras. Sigue estos pasos:

  • Genera un certificado SSL:
    • keytool -genkey -alias wildfly -keyalg RSA -keystore keystore.jks
  • Edita standalone.xml para agregar un conector HTTPS:
    • <connector name="https" protocol="HTTP/1.1" scheme="https"
secure="true" socket-binding="https"/>

4. Configuración del Firewall

Permite el puerto configurado en el firewall:

sudo ufw allow 8080/tcp

5. Inicio Automático

Configura WildFly como un servicio del sistema:

sudo nano /etc/systemd/system/wildfly.service

Añade el siguiente contenido:

[Unit]
Description=WildFly Application Server
After=network.target

[Service]
User=wildfly
Group=wildfly
ExecStart=/opt/wildfly/bin/standalone.sh
ExecStop=/opt/wildfly/bin/jboss-cli.sh --connect command=:shutdown
Restart=always

[Install]
WantedBy=multi-user.target

Luego, habilita el servicio:

sudo systemctl daemon-reload
sudo systemctl enable wildfly
sudo systemctl start wildfly
